トレーディングカード中古流通プラットフォーム向けの「倉庫入出庫シミュレーションツール」を Python で実装してください。
半機械化した倉庫の入庫・出庫・棚卸しアルゴリズムを差し替えながら、作業工数（コスト）と効率をシミュレーション・比較するためのツールです。

==================================================
1. 目的・全体像
==================================================
- 入庫／出庫／棚卸しのアルゴリズムを「シナリオ」として差し替え可能にする。
- 各シナリオを日次でシミュレーションし、作業コスト（秒）と「あたり率」を記録する。
- シナリオ単位で結果を保存し、Flask の簡易 Web アプリ（ローカル閲覧専用・非公開）で入力／シナリオ／出力を確認できる。
- グラフ表示は Web アプリから出力ファイルを参照し、HTML/CSS/JS で描画する想定。

==================================================
2. ドメインの前提
==================================================
- カードは入庫時に識別され、カードID（個体ID）が付与されて個体管理される。
- 個体情報は「商品ID（図柄）」と「ランク（状態）」を持つ。
- 出庫時は、同じ商品ID かつ 同じランク なら同一商品として扱い、どの個体を出庫してもよい。
- カードはストレージ（箱）に入る。ストレージは「ストレージID」を持ち、ストレージIDごとに収容上限枚数が設定される。

==================================================
3. 入力データ（CSV）
==================================================
データ置き場は既定で ./data/ 。入力は ./data/input/ 配下に CSV で置く。
以下のファイルとカラムを想定（カラム名はこの通りに実装してよい）。

- inbound.csv          … 入庫情報のリスト
    product_id, rank, card_id, arrival_date
- initial_stock.csv    … 初期在庫のリスト
    product_id, rank, storage_id, card_id
- storages.csv         … ストレージのリスト
    storage_id, capacity
- orders.csv           … 注文リスト
    order_date, product_id, rank, quantity

- 日付は YYYY-MM-DD 形式。
- シミュレーション期間は、入力データに現れる日付の最小〜最大から自動決定する（明示指定もできるとなお良い）。

==================================================
4. シミュレーションの流れ
==================================================
開始日から最終日まで、毎日以下を順に処理して日付を進める。

  1) その日の入庫情報（arrival_date が当日）のカードを、入庫アルゴリズムに従ってストレージへ入庫する。
  2) その日の注文（order_date が当日）に対し、出庫アルゴリズムで出庫するカードIDを選び、ストレージから出庫する。
  3) 棚卸しアルゴリズムを呼ぶ。実行するか否かはアルゴリズム側が日付を見て判断する。
     実行しない場合は「移動なし・コスト0」を返す。

毎日の作業内容（入庫・出庫・棚卸しの明細とコスト、あたり率など）をすべて記録する。

==================================================
5. アルゴリズムのインターフェース
==================================================
3種のアルゴリズムを Python クラスとして定義する。abstract base class（ABC）を用意し、
各シナリオの具象クラスがそれを継承する形にする。__init__ で任意の初期化引数を受け取れること。

- 入庫アルゴリズム
    引数: その日入庫するカードのリスト, 全ストレージの状態
    戻り値: [(card_id, 入庫先 storage_id), ...]

- 出庫アルゴリズム
    引数: その日出荷する注文のリスト, 全ストレージの状態
    戻り値: 各注文に割り当てたカードIDの一覧（注文と card_id の対応がわかる形。例: [(order, card_id), ...]）

- 棚卸しアルゴリズム
    引数: 日付, 全ストレージの状態
    戻り値: ( [(card_id, 移動先 storage_id), ...], かかったコスト秒数(float) )

「全ストレージの状態」は、各ストレージIDの収容上限・現在入っているカード（card_id, product_id, rank, 入庫日）を
アルゴリズムが参照できるデータ構造として渡すこと。状態オブジェクトは読み取り用とし、在庫の実更新は
シミュレーションエンジン側が戻り値に基づいて行う（容量超過などの不正な戻り値は検出してエラーにする）。

==================================================
6. シナリオ定義（YAML）
==================================================
シナリオは ./data/scenarios/{シナリオ名}.yaml に置く。
クラスのパス（モジュール:クラス名 など）と初期化引数、コスト単価を定義する。
importlib 等でクラスを動的ロードする。YAML 例:

  name: baseline
  inbound:
    class: "sim.algorithms.baseline:FillEmptyStorageInbound"
    args: {}
  outbound:
    class: "sim.algorithms.baseline:OldestFirstOutbound"
    args: {}
  stocktake:
    class: "sim.algorithms.baseline:NoStocktake"
    args: {}
  costs:
    inbound_per_storage_sec: 30      # ストレージID1つに入庫するのにかかる秒数
    outbound_pick_per_storage_sec: 60 # 出荷時にストレージID1つを持ってくる秒数
    outbound_per_card_sec: 2          # 出荷対象ストレージ内のカード1枚あたりの秒数（取出枚数に関係なく総数分かかる）

==================================================
7. 評価指標・コスト計算
==================================================
コストはすべて秒。

- 入庫コスト       = 入庫対象となったストレージIDの数 × inbound_per_storage_sec
- 出庫コスト       = 出庫対象となったストレージIDの数 × outbound_pick_per_storage_sec
                   + (出庫対象となった各ストレージ内にあったカード総数の合計) × outbound_per_card_sec
                   ※「対象ストレージに含まれるカード数」は、取り出す枚数ではなくそのストレージの在庫総数で課金する。
- 棚卸しコスト     = 棚卸しアルゴリズムが返した秒数。

あたり率（hit rate）:
  出庫のために割り当てたカード枚数 ÷ 対象となったストレージ内にあった総枚数。
  （ストレージ単位・日単位の集計、および全体集計ができること）

日次で「総工数（コスト内訳含む）」と「あたり率」を集計・記録する。

==================================================
8. 出力
==================================================
出力は ./data/output/{シナリオ名}/ 配下に置く。
- 日次の作業明細（入庫・出庫・棚卸しのログ）を CSV または JSON で保存。
- 日次サマリ（日付ごとの各コスト・総工数・あたり率）を、グラフ化しやすい形式（CSV/JSON）で保存。
- Web アプリのグラフがこれらのファイルを参照して描画できるようにする。

==================================================
9. 実行方法（CLI）
==================================================
- シナリオ名を引数に取ってシミュレーションを実行する CLI を用意する。
    例: python -m sim.run baseline
- データルートを切り替えられるオプション（既定 ./data/）も付ける。

==================================================
10. Flask Web アプリ（ローカル閲覧用）
==================================================
- 公開目的ではなくローカルで見るためのツール。
- 入力データ・シナリオ定義・出力結果を一覧／詳細表示できる。
- 日次の工数とあたり率のグラフ、および複数シナリオの比較表示ができる。
- グラフは Web アプリが出力ファイルを参照し、HTML/CSS/JS（クライアント側）で描画する。

==================================================
11. 最初に用意するシナリオ（実装確認用）
==================================================
動作確認のため、以下の baseline シナリオの具象クラス一式と、それを動かせるサンプル入力 CSV を用意する。
- 入庫: 空いているストレージIDにまとめて入庫する。
- 出庫: 古い入庫日のものから出庫対象にする。
- 棚卸し: 何もしない（常に移動なし・コスト0を返す）。
このシナリオをエンドツーエンドで実行し、出力と Web アプリ表示まで確認できる状態にすること。

==================================================
12. 実装上の要望
==================================================
- 言語は Python。uvを使う。
- ディレクトリ構成・モジュール分割（エンジン／アルゴリズム／IO／Web）を明確にする。
- 容量超過・存在しない card_id 指定・在庫不足などのバリデーションとエラーメッセージを入れる。
- README.md に、セットアップ・サンプル実行・シナリオの追加方法・Web アプリの起動方法を記載する。
- ARCHTECTURE.md に設計などの実装詳細を書く。
- 簡単な単体テスト（特にコスト計算とあたり率の集計）を用意する。
